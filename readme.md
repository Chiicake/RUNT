# RUNT 项目介绍

## 1. 项目概述

RUNT是一个基于Django框架开发的强化学习模型管理系统，提供完整的模型训练、管理、推理和训练指导功能。该系统旨在简化强化学习模型的开发流程，支持多种强化学习算法，为用户提供高效、便捷的模型管理解决方案。

## 2. 技术栈

| 类别 | 技术/框架 | 版本/说明 |
|------|-----------|-----------|
| 后端框架 | Django | Python Web框架 |
| 数据库 | MySQL | 关系型数据库 |
| 主要功能 | 强化学习模型管理 | 支持多种RL算法 |
| 开发语言 | Python | 主要开发语言 |

## 3. 核心功能

### 3.1 用户管理
- 用户登录验证
- 基于MySQL数据库的用户认证

### 3.2 模型管理
- 创建强化学习模型训练任务
- 支持多种强化学习算法（如DQN、PPO等）
- 模型训练状态监控
- 停止正在进行的训练任务

### 3.3 训练数据管理
- 查询模型列表，支持分页
- 查询特定模型的训练数据
- 训练回合奖励数据统计

### 3.4 模型推理
- 使用预训练模型执行推理
- 支持自定义初始观测值
- 返回推理结果（观测值、动作、奖励）

### 3.5 训练助手
- 基于训练超参数和奖励值提供专业训练指导
- 流式响应训练建议
- 超参数调整建议
- 训练策略优化建议
- 模型架构改进建议

## 4. 数据库设计

### 4.1 数据库配置

| 配置项 | 值 |
|--------|-----|
| 数据库类型 | MySQL |
| 主机地址 | localhost |
| 端口号 | 3306 |
| 用户名 | root |
| 密码 | 2512 |
| 数据库名 | runt |

### 4.2 主要数据模型

#### User（用户表）
- 用于存储系统用户信息
- 包含用户名和密码字段

#### RL Model（强化学习模型表）
- 存储模型基本信息和训练参数
- 包含算法名称、目标训练回合数、任务参数、计算资源参数、学习率等字段
- 记录模型训练状态

#### Training Data（训练数据表）
- 存储模型训练过程中的奖励数据
- 包含模型ID、训练回合、奖励值等字段

## 5. 快速开始

### 5.1 环境要求
- Python环境
- Django框架
- MySQL数据库

### 5.2 数据库初始化

1. 确保MySQL服务已启动
2. 创建名为`runt`的数据库
3. 在项目根目录执行以下命令生成迁移文件：
   ```bash
   python manage.py makemigrations
   ```
4. 执行迁移命令创建表：
   ```bash
   python manage.py migrate
   ```

### 5.3 创建初始用户

通过Django的shell创建初始用户：

```bash
python manage.py shell
```

在shell中执行以下命令：

```python
from web.models import User
user = User(username='admin', password='123456')
user.save()
```

### 5.4 启动服务

在项目根目录执行以下命令启动Django开发服务器：

```bash
D:\anaconda\envs\myrl\python.exe manage.py runserver
```

### 5.5 访问接口

服务器启动后，接口可通过以下地址访问：

- 登录：http://127.0.0.1:8000/login/
- 训练：http://127.0.0.1:8000/train/
- 停止训练：http://127.0.0.1:8000/stop-train/
- 模型列表：http://127.0.0.1:8000/models/
- 训练数据：http://127.0.0.1:8000/models/<model_id>/train-data/
- 模型推理：http://127.0.0.1:8000/testmodel/
- 训练助手：http://127.0.0.1:8000/trainingassistent/

## 6. 接口说明

### 6.1 主要接口列表

| 接口路径 | 方法 | 描述 |
|----------|------|------|
| `/login/` | POST | 用户登录验证 |
| `/train/` | POST | 创建强化学习模型训练任务 |
| `/stop-train/` | POST | 停止强化学习模型训练 |
| `/models/` | GET | 查询模型列表，支持分页 |
| `/models/<model_id>/train-data/` | GET | 查询特定模型的训练数据，支持分页 |
| `/testmodel/` | POST | 使用预训练模型执行推理 |
| `/trainingassistent/` | POST | 根据训练超参数和奖励值提供专业训练指导 |

### 6.2 接口特点

- 统一的响应格式，包含状态、消息和数据
- 支持JSON和表单数据格式
- 部分接口支持流式响应
- 支持分页查询
- 详细的错误信息返回

## 7. 项目架构

### 7.1 系统架构

RUNT系统采用典型的Django Web应用架构：

1. **前端**：通过HTTP请求与后端API交互
2. **后端**：基于Django框架，处理业务逻辑
3. **数据库**：MySQL存储模型和训练数据
4. **模型训练**：与强化学习算法模块交互

### 7.2 核心模块

- **认证模块**：处理用户登录验证
- **模型管理模块**：处理模型的创建、查询和状态更新
- **训练模块**：处理模型训练任务
- **推理模块**：处理模型推理请求
- **训练助手模块**：提供训练指导和建议
